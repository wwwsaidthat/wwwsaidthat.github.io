# Deep Graph Contrastive Representation Learning 

深度图对比表示学习

## 背景

1. 什么是图？

   图里面有点有边，点代表实体，边代表关系，图代表整个“环境” 

2. 什么是图表示学习？

   目的是为了**学习一个函数**，这个函数能够将图中的**节点**映射到向量上，同时还能表达出图的**属性和结构**的特征

   低维稠密向量：低维是指向量的维度不多，几十到几百左右；稠密是指向量中0很少，每位都有非0实数

3. 为什么需要图表示学习？

   1. 传统的CNN&Transformer没有办法处理这种**非欧、无序、稀疏**的图，CNN解决的是图片的识别，Transformer解决的是序列的识别，对于比如社交网络这种问题，这两个模型显然并不合适，因为他们都需要依赖**固定结构，固定邻居，有位置信息**的情况。

      1. 非欧：结构不规则、不对称、无固定网络，节点数量不固定，没坐标没顺序。

      2. 无序：在一个社交网络中并没有严格的顺序，并不能说谁在谁前面或者左边，它是无序的。

      3. 稀疏：一张图假设有$N$个点,对于完全图来说一共有
         $$
         \boxed{(N-1) + (N-2) + ... + 1 = \frac{N\cdot (N-1)}{2}}
         $$
         也就是$O(N^2)$的复杂度，对于$\boxed{边数 \ll 节点数^2}$就可以成为稀疏图。

   2. 因此才会产生所谓的GNN，也就是图神经网络

      1. 这个是先给每一个点都一个初始化，代表这个点对应的性质

      2. 然后不断的阅读周围的邻居，邻居阅读邻居的邻居，这样让每个点越看越远，然后就可以让整个图都在每个点上有所体现，这样就可以相对完整地在每一个节点的标识中考虑整个图的结构了     

         **“GNN可以看成是非欧图上的卷积”**

4. 图这类问题有什么下游任务？

   1. 可以从三个维度上说，点->边->图
      1. 节点分类：判断这个用户是不是机器人；这篇论文属于什么领域的；**dot**
      2. 链接预测：猜两个人是否会成为好朋友；猜测用户是否会购买某个商品；**edge**
      3. 图分类：判断这个分子是不是药物；判断这个电路是否有问题；**graph**
   2. 推荐系统--可以是点的任务，也可以是边的任务
   3. 知识图谱--点和边的构建（目的是让机器读懂关系）
   4. 交通预测--路口是点，道路是边，也是点和边级别的任务
   5. 蛋白质结构预测--图的任务

5. 传统的无监督图表示学习方法

   1. Skip-Gram
      1. Word2Vec 用来把单词变成「低维稠密向量」的无监督算法，Google2013年提出
         1. **核心思想：出现在相似上下文里面的意思就很相近**
         2. 由一对方法构成，他们都是word2vec的训练方法
            1. CBOW:用上下文预测中心词
            2. Skip-gram：用中心词预测上下文，这里的skip是在说相隔好几个词都会算在上下文里面，相隔几个词根据窗口大小人为规定
   2. DeepWalk & Node2Vec
      这两个并没有抛弃掉原始对于序列或者说是欧式的研究，想用多个序列来实现图的预测
      1. DeepWalk
         1. 首先是随机游走，得到很多的序列，相当于是得到了很多的语料，然后进行训练，出现在相同上下文的向量应该尽可能相似
         2. 训练的过程让经常一起出现的节点向量尽可能的相似
         3. 最后得到嵌入向量embedding
      2. Node2Vec
         1. 和DeepWalk的区别就在于第一步，他会用两个参数p和q来控制方向，p是回头的概率，q是向外探索的概率
         2. 都是生成很多的序列然后做embedding
      3. 局限性：
         1. 邻接矩阵：表示一个图中的点和点之间的联系的矩阵，哪个点和哪个点有联系，矩阵就有对应的数值上的表示
         2. 这种游走生成对应向量的方法本质上不是对于图的结构的模拟而是对于邻接矩阵的模拟
         3. 邻接矩阵少什么？
            1. 没有节点的性质
            2. 没有边的性质
            3. 没有全图的结构信息，只有两两的连通性看不出来图整体上的性质
            4. 而且这个结构非常的冗余低效
         4. 现有的无监督学习的方法会过度强调由网络结构定义的邻近信息，过度依赖邻接矩阵

6. 有标签的GNN训练过程中标签严重稀缺的问题很严重，需要大量的标注

7. 信息最大化原则InfoMax：满足已知约束下，让系统的不确定性（熵）或者输入-输出间的**互信息**达到最大

   1. 互信息MI Mutual Information

      1. 在不知道x的时候你对于y很不了解

      2. 知道x之后你对于y有了一定的了解

      3. 你对于y增加的了解就是x和y之间的互信息

         总结就是x告诉了你关于y多少信息

      4. 写成数学公式
         $$
         \boxed{I(x,y) = H(x)+H(y)-H(x,y)}
         $$

         1. 其中$I(x,y)$代表x和y共有的信息，H代表熵
         2. 如果互信息大于等于0，那么知道一个不可能增加另一个的不确定性
         3. 对称性x能告诉y的和y能告诉x一样多
         4. 对于几乎无关的两个东西，他们之间的互信息接近于0

   2. 也就是说信息最大化原则就是不增加乱说的约束条件，如果增加了一些本来不能确定的确定下来了，所以系统的熵就小了，输入和输出之间的互信息就会受到干扰，**很多深度学习表征学习的目标就是让编码/特征Z和输入X的信息尽可能大**

8. 正样本和负采样样本对比

   1. 所谓的正样本就是正向的样本，类似的样本

      1. 例如一个图片 裁剪 翻转 调色等得到的
      2.  一个句子 近义词替换 打乱顺序但是意思不变
      3. 训练的时候就是要告诉模型这两个样本长得像，向量要相近

   2. 所谓的负采样样本就是随机抽来的其他的样本，不同的样本

      1. 训练的时候就是要告诉模型这两个样本不像，向量要分开
      1. 负样本通过增强得到的是负样本

   3. $$
      \boxed{
      \mathcal{L} = -\log \frac{e^{\frac{q\cdot k_+}{\tau}}}{e^{\frac{q\cdot k_+}{\tau}}+\sum\limits_{i=1}^Ke^{\frac{q\cdot k_i^-}{\tau}}}
      }
      $$

   4. 负样本越像分母越大小数越小，loss也就越大

9. 深度图信息最大化 DGI

   1. 做法

      1. 思想是让每个节点的局部嵌入和整张图的全局摘要尽可能的相关，也就是互信息最大化
      2. 编码器encoder-GCN/GAT，输入特征+邻接矩阵，输出节点嵌入H
      3. Readout，对H做全局聚合，得到图级向量s（均值池化得到）
      4. 负样本生成，只打乱节点特征X的行，邻接矩阵A不变，得到坏图，编码得到嵌入H^~^
      5. 判别器D 判断两个嵌入H 和s之间的匹配关系是原配还是乱配
      6. 步骤
         1. 先用GNN得到节点的嵌入
         2. 然后通过读出函数readout得到全局摘要嵌入，也就是图嵌入
         3. 优化目标是**通过区分原始图中的节点与被破坏图中的节点**，最大化节点嵌入与图嵌入之间的互信息

   2. 缺点

      1. 这个等价于最大化输入节点特征和高层节点嵌入之间的互信息，没有考虑结构，只考虑了节点的原始特征，均值池化的读出函数不能保留节点级的嵌入中的独特特征，因此全局的图级向量也并不合理

         证明：现在考虑两个图，这两个图的图结构完全不同，只要满足这两个图中的节点特征均值是相同那么最后对应的s就是相同的，从而证明了跟图的结构根本就是无关的

      2. 对比学习的时候只在粗粒度层面上破坏节点的特征，而当矩阵稀疏的时候，不足以在破坏图中为节点生成不同的上下文

10. 无监督图表示学习对比框架GRACE **深度**图对比表示学习 这个深度就体现在真的考虑进去结构因素了

    1. 两种视图的生成方案：边删除和特征掩码，对边（结构层面，拓扑结构）和点（特征层面，节点属性）分别做了破坏 即边删除和特征掩码

    2. 效果非常好，在直推式任务的效果上 超过了有监督模型

       什么是直推式任务 -- 就是预测见过的有标准答案的东西

## 深度图对比表示学习

### 预备知识

1. 设图$\mathcal{G} = (\mathcal{V,E})$  
   1. 其中$\mathcal{V} = \{v_1,v_2,...,v_N\}$为节点集合
   2. 其中$\mathcal{E} \subseteq \mathcal{V} \times \mathcal{V}$ 为边的集合
2. 特征矩阵$X \in \mathbb{R}^{N \times F}$   邻接矩阵 $A\in \{0,1\}^{N\times N}$ 
   1. 每个节点的初始特征向量聚合在一起形成特征矩阵
   2. 其中F是指表示每个节点的特征向量一共有几维
   3. 其中N是指一共有多少个节点
   4. 当且仅当$(v_i,v_j)\in \mathcal{E}$ 才有$A(i,j)=1$
3. 我们的目标是学习一个图神经网络的编码器$f(X,A)$ 
   1. 输入是$X,A$ ，输出是低维度的节点嵌入向量
   2. 也就是说满足$F'\ll  F$
   3. 输出记为$H = f(X,A)$

1. ### 节点表示的对比学习框架详解

   1. 建立两张干扰图  $\mathcal{U}=f(\tilde{X_1},\tilde{A_1})$   $\mathcal{V} = f(\tilde{X_2},\tilde{A_2})$

   2. 对任意的一个节点$x_i$在两个图中对应的节点分别为$u_i$ 和 $v_i$ ,其中我们把$u_i$作为锚点，那么$v_i$就是正样本，一张图里面的其他的节点就是负样本就是$u_k u_j v_k v_j$这些都是负样本

   3. 定义一个$\theta{(u,v)} = s(g(u),g(v))$  其中s是余弦相似度，g是通过一个两层的多层感知机来实现的，将向量通过一个非线性投影增强一下表达能力，这是对比学习一个比较标准的步骤：

      1. 编码器encoder输出一个向量
      2. 进行两个多层感知机的处理使得向量的表达更有区分度，增加了非线性的函数，让表达更加丰富（如果直接做，对比学习的效果很差很差）
      3. 进行余弦相似度计算

   4. 成对损失公式如下：
      $$
      \boxed{
      \ell(u_i,v_i) = log(\frac{e^{\frac{\theta(u_i,v_i)}{\tau}}}{e^{\frac{\theta(u_i,v_i)}{\tau}} + \sum\limits_{k=1}^{N} \mathbb{1}_{[k\neq i]}e^{\frac{\theta(u_i,v_k)}{\tau}}+
      \sum\limits_{k=1}^{N} \mathbb{1}_{[k\neq i]}e^{\frac{\theta(u_i,u_k)}{\tau}}
      })
      }
      $$
      其中$\mathbb{1}$ 是一个函数，如果k和i相等，那么这个函数返回0，反之返回1

      这个$\tau$是一个温度参数

      负样本来自两个部分：同一张图的不同节点+不同图的不同节点 = 两张图各自除去对应节点的图

   5. 最终定义的整体的目标损失函数为
      $$
      \boxed{
      \mathcal{J} = \frac{1}{2N}\sum\limits_{i=1}^{N}[\ell(u_i,v_i)+\ell(v_i,u_i)]
      }
      $$

   6. 总结：

      ```python
      for epoch <- 1,2,3,... do
      	Generate two Graph
        using encoder get the node embeddings of two graphs
        compute the J function 
        Update parameters by applying stochastic gradient ascent to maximize J
      ```

      applying stochastic gradient ascent 这是随机梯度上升，每次从样本中取出来一部分样本计算梯度，然后沿着正方向更新参数，以最大化参数

2. ### 视图的生成

   1. 这个框架可以使用其他的破坏机制，下面我来阐述一下这个实验中作者所用的破坏机制

   2. 边删除（RE）remove edge
      $$
      \tilde{\mathbf{R}}_{ij}=
      \begin{cases}
      \sim \mathcal{B}(1-p_r)& A_{ij} = 1\\
      0 & A_{ij} = 0
      \end{cases}
      $$
      
      $$
      \boxed{\tilde{\mathbf{A}} = \mathbf{A}\circ\tilde{\mathbf{R}}}
      $$

   3. 节点特征掩码（MF）mask node features
      $$
      \tilde{m}_i \sim \mathcal{B(1-p_m)}
      $$

      $$
      \boxed{
      \tilde{\mathbf{X}} = [\mathbf{x}_1\circ\mathbf{\tilde{m}} ; \mathbf{x}_2\circ\mathbf{\tilde{m}},...,\mathbf{x}_N\circ\mathbf{\tilde{m}}]
      }
      $$

      这里的；就是拼接的意思

   4. 两个视图的生成依赖于两个超参数$p_r$ 和 $p_m$,两个视图，两组参数，各自生成各自的，在$p_r$ 和 $p_m$参数都小于等于0.8的时候模型对于这两个参数的选取并不敏感
